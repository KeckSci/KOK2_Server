# KOK2 Server V18.12.11 — 纯白副讯息解析器版

## 本版目标

在不修改客户端 EXE 的情况下，为登录欢迎语和怪物死亡提示实现真正的 `#FFFFFF` 纯白显示，同时保留统一的服务器消息配置。

## V18.12.9 为什么会显示标签

`0x0012 type=2` 的客户端格式器并不是简单地把正文拼到 `<font>` 后面，而是自动生成：

```text
<font color = #42FF00>正文</font><br>
```

其中：

- `0x07`：开启原文模式；
- `0x08`：关闭原文模式。

原文模式中的 `<font>` 会被当成普通文字，因此早期直接把 `</font><font ...>` 放入正文时，标签会原样显示并继续保持绿色。

客户端解析器中也明确存在对应状态切换：token `0x22` 将原文标志设为1，token `0x23` 将其设为0；字符表映射表明它们分别对应字节 `0x07` 和 `0x08`。

## 纯白实现

新的 `white` 样式仍使用已经验证可显示副讯息的：

```text
opcode = 0x0012
message_type = 2
sender = 空字符串
```

但服务器实际发送的正文为：

```text
</font><font color = #FFFFFF>实际文字
```

客户端副讯息格式器最终得到：

```text
<font color = #42FF00>
    
</font>
<font color = #FFFFFF>
    实际文字
</font>
<br>
```

含义是：

1. 立即退出客户端自动开启的原文模式；
2. 关闭自动添加的绿色字体；
3. 开启纯白字体；
4. 再次进入原文模式显示真正正文；
5. 在客户端自动追加 `</font><br>` 前退出原文模式。

两个连续的 `0x08` 都只是把同一状态设为0，因此不会产生额外文字或破坏状态。

聊天记录使用的另一套格式器没有自动包裹正文的 `0x07/0x08`，但同一个服务器正文仍会形成平衡的纯白结构，因此不会只修复副讯息而破坏聊天记录。

## 统一配置

`config.py` 继续统一管理所有服务器副讯息：

```python
ENABLE_SERVER_SIDE_MESSAGES = True

SERVER_SIDE_MESSAGES = {
    "login_welcome": {
        "enabled": True,
        "template": "欢迎光临万王之王2重启版",
        "style": "white",
    },
    "mob_death": {
        "enabled": True,
        "template": "{mob_name}无法抗拒死神的召唤，失去生命的光彩",
        "style": "white",
    },
    "mob_reward_gold": {
        "enabled": True,
        "template": "你找到了{gold}枚金币",
        "style": "system_light",
    },
    "mob_reward_experience": {
        "enabled": True,
        "template": "你得到了{experience}点经验",
        "style": "green",
    },
}
```

现有样式：

| style | 显示颜色 | 协议路径 |
|---|---|---|
| `white` | `#FFFFFF` | `0x0012 type=2` + 客户端原文模式切换 |
| `system_light` | `#FDF597` | `0x0001` 客户端固定颜色 |
| `green` | `#42FF00` | `0x0012 type=2` 客户端固定颜色 |

配置模板中仍禁止直接写 `<font>` 标签以及 `0x07/0x08` 控制字符。纯白控制序列只由协议适配器生成，避免普通配置破坏客户端富文本状态。

## 检查结果

- 全项目 Python 语法检查通过；
- 所有主要模块导入通过；
- 中文 CP936 编码通过；
- `white` 消息 wire payload 中确认包含正确的 `0x08 / 0x07 / 0x08` 顺序；
- 重建客户端副讯息格式和聊天记录格式后，两者的 `<font>` 与原文模式均保持平衡；
- 配置模板不能注入客户端控制字符或字体标签；
- `kok2.db` 未修改。

## 实机验证重点

本路径具有完整静态代码依据，但仍需客户端实机确认：

1. 欢迎语是否显示为真正纯白；
2. 是否完全不显示 `</font>` 等标签；
3. 怪物死亡提示后，金币和经验颜色是否仍分别保持浅金色与绿色；
4. 多次连续击杀后，后续消息颜色是否没有被污染。
